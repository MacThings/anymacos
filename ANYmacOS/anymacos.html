New in 1.1.3

  • It is now possible to create an installer application without SIP disabled. This is no longer created directly in "/Applications" but in a sparse image which is created, mounted and filled by ANYmacOS in the download path stored by the user. This has the advantage that you don't have to write to a system protected directory (/Applications) anymore. You end up with a volume named "Installer-App" on your desktop where the installer application is located.
